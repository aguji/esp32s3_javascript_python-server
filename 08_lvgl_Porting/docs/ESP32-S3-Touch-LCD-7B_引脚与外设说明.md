# ESP32-S3-Touch-LCD-7B 原理图怎么看 & 哪些引脚可接外设

## ★ 只接官方触摸屏 + ESP32-S3 时（无 SD/RS485/CAN）

**结论：可以。** 板子上没接 SD 卡、RS485、CAN 等时，这些外设在原理图上占用的引脚**没有被实际使用**，你可以在程序里当普通 GPIO 用。

- **你当前实际占用的**只有：**LCD（RGB + 同步）+ 触摸 I2C + 触摸复位用的 GPIO4**。
- 下表里“仅被 SD/RS485/CAN 占用”的引脚，在你没接这些模块时**都可以用**。

---

## 一、怎么从原理图里“找能用的引脚”

### 1. 先找主芯片（ESP32-S3）
- 原理图里会有一个大框图，标 **U8 ESP32-S3-WROOM-1-N16R8**。
- 周围会列出所有从芯片引出的引脚：**IO0~IO21、IO38~IO48、3V3、GND、EN、TXD0、RXD0** 等。

### 2. 看“引脚分配表”或“连接关系”
- 原理图里通常有一张表或一块图，写 **LCD / SD / WROOM** 或 **GPIO / Sensor** 等，表示：**哪个 GPIO 接到了板子上的什么功能**。
- 例如：
  - **IO3 → VSYNC**：说明 IO3 已经给 LCD 用，**不能再接你的外设**。
  - **IO19、IO20 → GPIO**：说明这两个是“普通 GPIO”，**可以接外设**。

### 3. 记住一个原则
- **已经接到 LCD、触摸、SD、RS485、CAN、USB 的引脚 = 已被占用，不要接外设。**
- **标成 GPIO、EXIO、或单独引到排针/插座的引脚 = 可以接外设。**

## 二、本板（7B）引脚占用情况（从原理图整理）

### 你当前工程“一定在用”的引脚（只接屏+触摸时，不要占这些）

| 功能 | 占用引脚 | 说明 |
|------|----------|------|
| LCD RGB | IO3, IO5, IO7, IO46, IO0,1,2,10,14,17,18,21,38,39,40,41,42,45,47,48 | 显示屏 VSYNC/DE/PCLK/HSYNC + DATA0~15 |
| 触摸 I2C | **IO8(SDA), IO9(SCL)** | GT911 触摸 |
| 触摸复位 | **IO4** | 代码里 touch reset 用到 |

以上这些**不能**再接你的外设。

### 原理图上给“板载外设”留的引脚（你没接就可以用）

| 功能 | 原理图占用引脚 | 你没接该外设时 |
|------|----------------|----------------|
| SD 卡 | 多为 IO11, IO12, IO13 等 SPI | **可当 GPIO 用** |
| RS485 | IO15(TX), IO16(RX) | **可当 GPIO 或 UART 用** |
| CAN | 由外接芯片接出 | 若未接，对应脚也可用 |
| USB/下载 | 43/44 等 | 建议保留下载，不接外设 |

结论：**没接 SD/RS485/CAN 时，IO6、IO11、IO12、IO13、IO15、IO16、IO19、IO20 等都可以接你的外设。**

## 三、可以接外设的引脚

### 0. 只接屏+触摸时，可直接用的 GPIO（推荐）

你当前工程只用了 LCD + 触摸，下面这些脚**没有被占用**，且你没接 SD/RS485 时也未被板载电路占用，**可以直接当 GPIO 用**：

| GPIO | 说明 |
|------|------|
| **IO6**  | 可用 |
| **IO11** | 多用于 SD MOSI，无 SD 则可用 |
| **IO12** | 多用于 SD MISO，无 SD 则可用 |
| **IO13** | 多用于 SD SCK，无 SD 则可用 |
| **IO15** | 多用于 RS485 TX，无 RS485 则可用 |
| **IO16** | 多用于 RS485 RX，无 RS485 则可用 |
| **IO19** | 可用 |
| **IO20** | 可用 |

**注意：** IO43、IO44 多为 USB/下载口，建议保留，不要接外设以免影响烧录。

### 1. 扩展座上的 EXIO（若板子有引出）

原理图上有 **EXIO0 ~ EXIO7**，通过电平转换/保护接到排针，适合接外设：

| 排针名称 | 说明 | 注意 |
|----------|------|------|
| **EXIO0** | 通用 GPIO，部分板子还支持 ADC/PWM/RESET | 确认板子标注是否带 ADC/PWM |
| **EXIO1** | 通用 GPIO（有的板子连 CTP_RST/DISP） | 看板子背面或丝印是否引出 |
| **EXIO2** | 通用 GPIO（有的板子连 LCD_RST） | 同上 |
| **EXIO3** | 通用 GPIO | 可接传感器、继电器等 |
| **EXIO4** | 通用 GPIO（有的板子连 SDCS） | 若不用 SD 卡，可能可当普通 IO |
| **EXIO5** | 通用 GPIO（有的板子连 USB_SEL） | 看原理图是否真正引出 |
| **EXIO6** | 通用 GPIO（有的板子连 LCD_VDD_EN） | 同上 |
| **EXIO7** | 通用 GPIO | 可接外设 |

- **使用前**：用万用表或看板子背面丝印，确认 EXIOx 是否真的引到排针，以及是否和 CTP_RST、LCD_RST 等复用。
- 若原理图上有 **EXIO_ADC、EXIO_PWM**，则扩展座上可能有**模拟输入**和**PWM 输出**，可接模拟传感器、舵机等。

### 2. 标成“GPIO”的芯片引脚

从原理图标注可知：

| 引脚 | 说明 |
|------|------|
| **IO19** | 标注为 GPIO，可接外设（注意电平 3.3V） |
| **IO20** | 标注为 GPIO，可接外设 |
| **IO6** | 标注为 GPIO6，在 J8 等位置引出，可接外设 |

### 3. 电源与地

- **3V3**、**GND**：扩展座和 J8/J9 上一般都有，给外设供电。
- **5V**：部分排针有 5V（如 Type-C 输入后稳压前），接大电流或 5V 外设前要看原理图电流能力。

## 四、如何在你自己的原理图里“对号入座”

1. **打开 PDF**，用“查找”（Ctrl+F）搜：
   - **EXIO** → 看 EXIO0~7 连到哪颗芯片、是否到排针。
   - **GPIO** 或 **IO19、IO20、IO6** → 看是否只做普通 IO 且引出。
2. **看排针/插座**：找 **H3、J8、J9** 等，旁边会标 1、2、3… 和 3V3、GND、EXIOx、GPIO6 等，对应表里“可接外设”的脚。
3. **看芯片 U10**（若有）：很多板子用 U10 做电平转换或扩展，EXIO0~7、EXIO_ADC、EXIO_PWM 会从 U10 连到排针，这样你就知道**哪个排针对应哪个 EXIO/功能**。

## 七、RS485 的 A、B 在原理图里怎么找（对应板子哪个口）

### 1. RS485 的 A、B 是什么

- **A**、**B** 是 RS485 的**两根差分线**：A = 正相（非反相），B = 负相（反相）。外接设备时，**A 接 A、B 接 B**（反接会通信异常）。
- 板子上的“接口”就是**接线端子或排针**，原理图里会标出哪一脚是 A、哪一脚是 B。

### 2. 在原理图里怎么找“A 对应哪个口”

按下面顺序看，就能对上“芯片上的 A/B”和“板子上的接口”：

**第一步：找到 RS485 收发芯片**

- 用 PDF 的 **Ctrl+F** 搜：**RS485**、**SP3485**、**MAX3485**、**485** 等。
- 常见型号：**SP3485EN**、MAX3485 等，图里会有一个小方框（芯片），旁边标引脚名。

**第二步：在芯片上找到 A、B 两个引脚**

- 芯片引脚表里会有 **A**、**B**（有的还标成 **Y/Z** 或 **A/B**）。
- 例如 SP3485EN：通常 **Pin 6 = A**，**Pin 7 = B**（以芯片 datasheet 为准）。原理图里这两个脚会连到一根“线”（网络），并带**网络名**，如 `RS485_A`、`RS485_B` 或直接标 `A`、`B`。

**第三步：顺着线找到板子接口**

- 从芯片的 **A**、**B** 脚顺着线（同名网络）往“外”找：
  - 会经过**保护器件**（如 TVS、电阻）、**端子/排针**（如 J6、J7、H2、CON2 等）。
- 在**接线端子或排针**的符号旁，原理图会标 **1、2、3…** 或 **A、B、GND**：
  - 连到芯片 **A** 的那一脚 = 板子上的 **A 口**  
  - 连到芯片 **B** 的那一脚 = 板子上的 **B 口**

**第四步：对一下实物板子**

- 板子丝印或接口旁边通常会标 **A**、**B**、**485**、**RS485** 等。
- 若只有 **1、2**：看原理图里该 connector 的 **1 = A 还是 B**，对应到丝印的 1、2 即可。

### 3. 小结（怎么看出 A/B 对应什么口）

| 看哪里 | 看什么 |
|--------|--------|
| 原理图里 RS485 芯片 | 找到标 **A**、**B** 的引脚 |
| 同页或跨页 | 这两根线连到哪个 ** connector（Jxx/Hxx）** 的哪一脚 |
| 该 connector 的引脚表 | 例如 “1 = A, 2 = B” 或 “A, B, GND” |
| 板子实物 | 丝印或接口旁的 **A/B** 或 **1/2** 对应关系 |

**一句话**：原理图里 **RS485 芯片的 A 脚 → 连到哪个 connector 的哪一脚**，那一脚就是板子上的 **A 口**；B 同理。
