# 扫码登录对接说明（网页端 + ESP32 + Python 服务器）

## 流程概览

1. **设备（ESP32 或网页）** 向 Python 服务器请求一个唯一 **token**（`GET /request_token`）。
2. **设备** 生成二维码，内容为：`http://服务器IP:8000/login?token=xxx`。
3. **用户用手机扫码** → 浏览器打开该 URL → 服务器将对应 token 状态改为 `success`，并返回“登录成功”页面。
4. **设备** 每隔 2 秒轮询 `GET /check_token?token=xxx` → 若返回 `success`，则登录成功（ESP32 切换界面，网页端写 localStorage 并跳转主菜单）。

这样每个设备的二维码都带唯一 token，服务器能区分是哪一个设备在请求登录。

---

## 一、Python 服务器（本机桥梁）

- 代码位置：项目根目录 **`server.py`**。
- 使用 Python 内置 `http.server`，无需安装第三方库。

### 运行方式

```bash
# 在项目根目录执行
python server.py
```

- 监听：`0.0.0.0:8000`（局域网可访问）。
- 启动后会提示本机 IP，例如：`http://192.168.1.100:8000`。
- 请确保防火墙放行 8000 端口；手机、ESP32、电脑需在同一局域网。

### 接口说明

| 路径 | 说明 |
|------|------|
| `GET /request_token` | 设备请求 token，返回纯文本 token 字符串。 |
| `GET /check_token?token=xxx` | 设备轮询，返回 `pending` 或 `success`。 |
| `GET /login?token=xxx` | 手机扫码打开的页面，确认后将该 token 置为 `success`。 |

---

## 二、网页端（本仓库）对接

- 页面：**`pages/user/login-qrcode.html`**。
- 行为由 **`window.SCAN_LOGIN_SERVER`** 决定：
  - **留空 `''`**：演示模式，约 5 秒后自动用 mock 用户登录。
  - **设为服务器地址**（如 `'http://192.168.1.100:8000'`）：向该服务器请求 token，显示真实二维码，并轮询状态；扫码成功后写 localStorage 并跳转主菜单。

### 配置方式

在 `login-qrcode.html` 中找到：

```javascript
window.SCAN_LOGIN_SERVER = '';
```

改为你的 Python 服务器地址（与 ESP32 使用同一地址即可），例如：

```javascript
window.SCAN_LOGIN_SERVER = 'http://192.168.1.100:8000';
```

- 网页需通过 **http** 打开（如 `http://192.168.1.100:8080/...` 或本机静态服务），以便与服务器同网段访问。
- 若浏览器报跨域错误，请确认 `server.py` 已对相关接口返回 `Access-Control-Allow-Origin: *`（当前代码已包含）。

## 三、ESP32 端（你本地工程）对接要点

- **服务器地址**：与网页端一致，例如 `http://192.168.1.100:8000`。
- **流程**：
  1. 启动/进入扫码界面时：`GET /request_token` 拿到 token。
  2. 生成二维码内容：`http://192.168.1.100:8000/login?token=<token>`。
  3. 定时（如每 2 秒）请求 `GET /check_token?token=<token>`。
  4. 若返回 `success`，则登录成功，切换界面（如 `lv_scr_load(scr_main)`）。

你提供的 ESP32 代码（`requestToken()`、`checkLoginStatus()`、`updateQRCode()`、轮询定时器）按上述接口实现即可，无需再改服务器或网页逻辑。

---

## 四、测试步骤

1. 在本机运行 **`python server.py`**，记下控制台打印的局域网 IP（如 `192.168.1.100`）。
2. **网页端**：在 `login-qrcode.html` 中设置 `SCAN_LOGIN_SERVER = 'http://192.168.1.100:8000'`，用浏览器打开该页，应看到二维码和“请用手机扫描”提示。
3. **手机**：用浏览器或扫码工具扫该二维码，应打开“登录成功”页面。
4. **网页**：约 2 秒内自动跳转到主菜单。
5. **ESP32**：将代码中的服务器 IP 改为同一地址，上传后屏幕显示二维码，手机扫码后 ESP32 轮询到 `success` 并切换界面。

---

## 五、补充说明

- **token 存储**：当前为内存字典，重启服务器会清空；毕设演示足够。若需持久化可改用 SQLite 等。
- **二维码有效期**：可在服务器端对超过 2 分钟的 `pending` token 做清理；ESP32/网页也可在超时后重新请求 token 并刷新二维码。
- **与 main-menu 的关系**：`main-menu.html` 仍按原有逻辑根据 `localStorage` 的 `isLoggedIn`、`currentUser` 显示；扫码成功后由 `login-qrcode.html` 写入这两项并跳转，因此无需修改 main-menu。
