# 智能图书馆导览机器人系统 — 用户端功能说明（论文用）

本文档从系统架构、功能模块、交互流程和技术实现四方面对用户端进行详细介绍，便于写入毕业论文的“系统设计与实现”或“用户端设计”等章节。

---

## 一、用户端总体架构

### 1.1 定位与入口

用户端面向图书馆读者，提供图书检索、自助借还指引、个人借阅管理、馆内导航及帮助服务。系统采用**多页面（多页版）**与**单页面（单页版）**并存的架构：

- **多页版**：以根目录 `index.html` 为站点首页，用户从首页可进入 `pages/user/login.html` 登录，登录后进入 `pages/user/main-menu.html` 主菜单，再通过链接进入各功能页面。每个功能对应独立 HTML 页面，通过 `window.location.href` 进行页面跳转。
- **单页版**：根目录 `index.html` 内嵌欢迎页、主菜单、检索、借书、还书、个人中心、导航、帮助等多个区块（`<section>`），通过 JavaScript 的 `showPage(pageId)` 在同一页面内切换显示内容，无整页跳转，适合作为终端一体机界面。

论文描述时可按实际采用的版本（如“本系统用户端采用多页面架构，便于维护与扩展”）进行说明。

### 1.2 页面与路由结构

用户端页面均位于 `pages/user/` 目录下，与根目录 `index.html` 的关系如下：

| 路径 | 说明 |
|------|------|
| `index.html` | 站点首页：欢迎页 + 单页版功能集合（可选） |
| `pages/user/login.html` | 登录方式选择页（RFID / 扫码 / 账号密码） |
| `pages/user/password-login.html` | 账号密码登录表单页 |
| `pages/user/login-rfid.html` | RFID 刷卡登录页（模拟） |
| `pages/user/login-qrcode.html` | 扫码登录页（模拟） |
| `pages/user/main-menu.html` | 登录后主菜单（多页版入口） |
| `pages/user/search.html` | 图书检索 |
| `pages/user/self-service.html` | 自助借还说明与流程指引 |
| `pages/user/profile.html` | 个人中心（借阅统计、当前借阅、借阅历史） |
| `pages/user/navigation.html` | 导航指引（楼层平面图、书籍位置查询） |
| `pages/user/help.html` | 帮助与支持（使用指南、联系方式、系统设置） |

多页版下，未登录用户直接访问 `main-menu.html` 会被重定向至 `index.html`，保证入口一致性与权限控制。

### 1.3 公共界面元素

各用户页面均包含统一的**顶部状态栏**（`header.status-bar`），用于展示：

- **左侧**：网络状态（如“已连接”）、电量（如“100%”）；
- **中间**：当前时间（每分钟更新）；
- **右侧**：当前登录用户姓名或“未登录”。

状态栏通过公共样式文件 `css/base.css` 及脚本 `js/system.js`、`js/auth.js` 中的 `updateUserStatus()`、`updateTime()` 等实现，保证全站体验一致，并便于在论文中描述为“统一的顶栏与状态反馈”。

## 二、功能模块详细说明

### 2.1 首页与欢迎页

- **文件**：根目录 `index.html`（欢迎区块 `#welcome-page`）。
- **功能**：
  - 展示系统名称“智能图书馆导览机器人”及简要欢迎语；
  - **登录使用**：跳转至 `pages/user/login.html`；
  - **游客能力**：未登录时可“查看图书馆布局”“搜索图书位置”，对应单页版内的导航页与检索页；
  - 语言切换：中文 / English（按钮预留，可扩展）。
- **设计要点**：首页作为唯一入口时，不根据本地存储的登录状态自动跳转主菜单，始终先展示欢迎页，避免在公共终端上误显示上一用户的会话，适合写入论文的“安全与隐私保护”或“用户体验”小节。

### 2.2 登录模块

#### 2.2.1 登录方式选择（login.html）

提供三种登录入口：

1. **RFID 刷卡登录**：跳转 `login-rfid.html`，模拟校园卡靠近读卡器；
2. **扫码登录**：跳转 `login-qrcode.html`，模拟手机 APP 扫描二维码；
3. **账号密码登录**：跳转 `password-login.html`，输入学号/工号与密码。

三种方式在论文中可归纳为“多模态身份认证”，体现对传统刷卡、移动端扫码与账号密码的兼容设计。

#### 2.2.2 账号密码登录（password-login.html）

- **输入项**：账号（学号/工号）、密码；
- **校验逻辑**：在全局用户数据 `AppState.mockUsers` 中查找 `id === 账号` 且 `password === 密码` 的用户；若存在则写入 `localStorage`（`isLoggedIn`、`currentUser`），并跳转：
  - 普通用户 → `main-menu.html`；
  - 管理员（`role === 'admin'`）→ `pages/admin/admin-menu.html`。
- **反馈**：错误时在页面内显示“账号或密码错误”，并清空密码框、恢复登录按钮，便于在论文中描述为“即时错误反馈与防重复提交”。

#### 2.2.3 RFID / 扫码登录（模拟）

- **login-rfid.html**：用户点击“开始识别”后，延迟约 2 秒模拟读卡成功，将预设用户（如学号 20230001）写入 `localStorage` 并跳转主菜单。
- **login-qrcode.html**：页面加载后约 5 秒自动模拟扫码成功，同样写入本地用户并跳转主菜单。

论文中可说明：RFID 与扫码为仿真流程，实际部署时可对接硬件读卡器与扫码服务，前端交互与状态管理保持一致。

### 2.3 主菜单（main-menu.html）

- **作用**：登录后多页版的中心枢纽，提供各功能入口与账号相关操作。
- **展示内容**：
  - 欢迎语及当前用户名（由 `updateUserStatus()` 写入 `#logged-user`）；
  - 管理员登录时显示“管理后台”入口及提示文案；
  - 功能按钮：图书检索、自助借还、个人中心、导航指引、帮助；管理员另有“管理后台”按钮；
  - **密码重置**：弹窗内输入当前密码、新密码、确认新密码，校验通过后更新 `AppState.mockUsers` 中对应用户的 `password`，并同步至 `localStorage`（`library_mockUsers`、`currentUser`）；
  - **退出登录**：清除 `isLoggedIn`、`currentUser`，跳转 `index.html`。
- **权限与安全**：未登录访问主菜单时，脚本检测 `localStorage.isLoggedIn`，若未登录则重定向至 `index.html`，保证“登录后可见”的访问控制，适合在论文中作为“前端权限与会话管理”的实例。

### 2.4 图书检索（search.html）

- **功能**：按关键词与条件查询馆藏，并展示结果列表。
- **输入与筛选**：
  - 搜索框：支持书名、作者、ISBN（占位示例：人工智能导论、刘伟、9787302455106）；
  - 筛选条件：分类（计算机、文学、科学、历史、艺术、经济）、状态（可借阅/已借出/已预约）、排序（相关性、书名、作者、出版年份）。
- **检索逻辑**（`js/search.js`）：在 `AppState.mockBooks` 中进行前端过滤，匹配书名、作者、ISBN 及上述筛选条件，结果写入 `#results-list`，并更新“共 X 条结果”。
- **展示内容**：每条结果包含封面色块、书名、作者、ISBN、架位、借阅状态（可借阅/已借出）；支持清空条件并重新搜索。
- **论文可写**：检索模块采用“前端过滤 + 多维度筛选”的设计，便于在无后端时演示；实际部署可替换为向后端 API 的请求，界面与交互保持不变。

### 2.5 自助借还（self-service.html）

- **定位**：说明性页面，不执行实际借还操作，而是引导用户到馆内自助借阅机完成操作。
- **内容**：
  - 自助借阅机位置（如一楼大厅、二楼阅览区东侧）；
  - 借书流程：选择借书 → 刷校园卡或输入学号/工号 → 扫描书籍条形码 → 确认借阅；
  - 还书流程：选择还书 → 放入还书口或扫描条形码 → 等待系统确认；
  - 人工服务：设备故障或疑问时可至一楼服务台办理。
- **设计意义**：将“导览/查询”与“线下借还”分离，用户端侧重信息展示与流程说明，实际借还由馆内终端与后台完成，论文中可归纳为“线上线下融合的服务边界划分”。

### 2.6 个人中心（profile.html）

- **用户信息区**：头像、姓名、学号/工号、院系（由 `AppState.currentUser` 及 `updateUserInfo()` 等填充）。
- **统计指标**：当前借阅册数、历史借阅总数、信用积分，用于快速了解个人借阅状况。
- **当前借阅书籍**：列表展示当前在借图书（书名、作者、应还日期等），数据来自 `AppState.currentUser.borrowedBooks` 与 `AppState.mockBooks` 的关联。
- **辅助功能**：
  - **借阅历史**：进入同页内另一区块（`#borrow-history-page`），按时间倒序展示历史借还记录（借阅日期、应还日期、归还日期、状态：借阅中/已归还）；
  - **设置**：预留入口，可扩展字体、语音、高对比度等（与 help 页设置一致）。
- **数据来源**：用户数据与借阅记录来自 `AppState.mockUsers` / `AppState.currentUser`，若本地存在 `library_mockUsers` 则优先使用，保证管理员代录、自助借还等操作后的数据在各页面一致，可在论文中说明为“以全局状态与本地持久化为基础的个人中心数据一致性”。

### 2.7 导航指引（navigation.html）

- **楼层选择**：一至四楼按钮，切换当前楼层平面图。
- **平面图**（`js/navigation.js`）：使用 SVG 动态绘制当前楼层的简化平面图，包括书架区域（A–F 区）、图例（当前位置、目标位置、书架区域），以及“当前位置”与“目标位置”的标记点。
- **书籍位置查询**：输入书名或 ISBN，在 `AppState.mockBooks` 中查找对应图书的 `location` 字段，生成文字版导航步骤（如从当前位置出发 → 前往某区 → 到达某架位），并在平面图上标出目标位置点。
- **论文可写**：导航模块将“馆藏位置数据”与“楼层平面可视化”结合，实现“查书—定位—指引”的一体化，便于读者快速找到目标书架，体现导览机器人的空间信息服务能力。

### 2.8 帮助与支持（help.html）

- **使用指南**：借书、还书的步骤说明（与 self-service 流程一致）。
- **联系我们**：服务热线、邮箱、服务台地址。
- **系统设置**：字体大小（小/中/大）、语音提示开关、高对比度模式开关，用于无障碍与个性化，设置可写入全局状态或本地存储并在各页生效。

## 三、用户端交互流程小结

1. **入口**：用户打开 `index.html`，看到欢迎页；点击“登录使用”进入 `login.html`。
2. **认证**：选择 RFID/扫码/账号密码之一，完成认证后写入 `localStorage` 并跳转主菜单（或管理员后台）。
3. **主菜单**：从主菜单可进入检索、自助借还说明、个人中心、导航、帮助；可修改密码、退出登录。
4. **检索**：输入关键词并设置筛选条件，查看结果列表及架位、状态。
5. **自助借还**：阅读说明与流程，按指引到馆内自助机操作。
6. **个人中心**：查看个人信息、当前借阅、借阅历史及统计。
7. **导航**：选择楼层查看平面图，输入书名/ISBN 获取架位与文字导航。
8. **帮助**：查阅使用说明、联系方式和系统设置。

上述流程可在论文中整理为“用户端业务流程图”或“用例—页面对照表”。

## 四、技术实现要点（供论文“实现”章节引用）

### 4.1 前端技术栈

- **结构**：HTML5，语义化标签与无障碍友好 id/class。
- **样式**：CSS3，集中变量（`:root` 中的颜色、间距、字体等）在 `base.css`，各页面专用样式在 `search.css`、`profile.css`、`navigation.css`、`help.css`、`menu.css`、`login.css` 等。
- **行为**：原生 JavaScript（ES5/ES6 兼容写法），按功能拆分为 `app-state.js`、`auth.js`、`utils.js`、`notification.js`、`system.js`、`search.js`、`profile.js`、`navigation.js` 等，便于维护与分工。

### 4.2 状态与数据

- **全局状态**：`AppState`（`app-state.js`）集中管理当前用户、登录标志、模拟用户列表、模拟馆藏等；页面加载时从 `localStorage` 恢复 `currentUser`、`isLoggedIn`，必要时用 `mockUsers` 补全用户信息（如借阅历史）。
- **持久化**：登录状态与用户对象存于 `localStorage`（`isLoggedIn`、`currentUser`）；管理员代录、借还等会更新 `library_mockUsers`、`library_mockBooks`，下次加载时写回 `AppState`，实现多页面、多角色下的数据一致。

### 4.3 安全与体验

- **权限**：主菜单等需登录页面在加载时检查 `localStorage.isLoggedIn`，未登录则重定向首页；管理员功能通过 `role === 'admin'` 控制入口显示。
- **输入安全**：动态插入内容时使用 `escapeHtml()` 等工具，降低 XSS 风险。
- **反馈**：操作结果通过 `showNotification(message, type)` 在顶栏或固定区域提示，类型包括 info/success/warning/error，提升可读性与可访问性。
